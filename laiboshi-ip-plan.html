第二阶段：表达体系

EXPRESSION SYSTEM — "他怎么说话、视频长什么样"
核心原则

定位 → 表达 → 内容 → 信任 → 变现。表达不稳定 → 用户记不住你 → 定位再准也白费。表达不是"教他怎么说"，而是"固定他怎么呈现"。

2.1 赖博士的表达基因（不可改，只能放大）

基因表现策略
边说边想学者型口语表达，句子长、自然流畅，不是背稿保留真实语感 后期精简重复和跑题部分
学者思维喜欢从原理讲起，先铺垫再展开后期把最炸的一句话提到开头当钩子
指实物站在鸭场里，边说边指藻池/鸭蛋/水管鼓励多拍。比干讲强10倍
客家口音普通话带客家腔保留。这是地域身份标识，不是瑕疵
技术自信讲原理时不犹豫，数据张口就来这就是IP壁垒。竞品抄不走

2.2 表达六固定（每期保持一致）

固定项规范
开场方式永远不自我介绍。第一句就是钩子。禁用"今天给大家分享…"
说话节奏赖博士自然节奏为主。后期用B-roll打断过长段落，制造呼吸感
画面语言赖博士出镜60% + B-roll（鸭蛋切开/鸭场/藻池/水培菜）40%
标题语言永远是一个问题/一个反常识/一个数字。禁用纯话题标签
结尾动作每次只做一个引导——收藏/评论/来看，不同步做两个以上
视觉识别封面统一模板 + 片尾LOGO+Slogan + 字幕关键词黄绿色高亮

2.3 表达节奏模板（90秒视频示例）

0-3秒 钩子（赖博士原声或配音字幕，一句话炸开） 3-8秒 赖博士出镜，接住钩子展开讲 8-25秒 第一层内容（他讲，中间切1-2次B-roll） 25-30秒 小停顿/转场（B-roll或字幕过渡） 30-55秒 第二层内容（继续讲，再切1-2次B-roll） 55-75秒 第三层内容或案例 75-85秒 总结句（赖博士原声金句） 85-90秒 结尾引导 + 片尾落版

⚠️ 这是后期剪辑的节奏模板。赖博士自由讲，我们按这个节奏剪。

2.4 表达质量自检（每条发布前）

#检查项通过标准
1前3秒有钩子吗？不看标题也知道这条要讲什么
2钩子是反常识/问题/数字吗？不是"今天给大家分享…"
3有B-roll吗？至少3处画面切换
4标题能让人想点吗？不超过25字，不是话题标签
5结尾有单一CTA吗？收藏/评论/来看 只做一样
6表达节奏对吗？真实自然，但不拖沓让人想划走

赖博士现有视频风格诊断

基于40条视频文案分析

现有风格画像

维度发现判断
形式口播——站在鸭场/藻池/面馆里对着镜头讲，偶尔指身边实物✅ 方向正确
语言大量"呢""呃""就是说""实际上呢"，句子很长，边说边想✅ 真实，保留
话题无抗NAE / 藻工厂 / DHA圈纹 / 水循环 / 连城白鸭 / 展会✅ 就是他该讲的
逻辑学者思维——先铺垫再展开，喜欢从原理讲起✅ 是他的壁垒
数据有一条到6.5万浏览（"欢迎大家来鸭场"）⚠️ 证明方向能爆
标题用话题标签当标题，没有点击欲望❌ 最大硬伤
开头没有钩子，直接开始讲，用户前3秒不知道要讲什么❌ 完播率杀手
结尾没有互动引导，讲完就结束❌ 零互动转化
剪辑基本一刀不剪或剪很少，保留了全部自然表达❌ 内容扎实但节奏偏慢
核心结论

赖博士不是不会做内容——他已经在做自己想做的事了。问题不是"讲什么"，是"怎么包装"。

他负责：提供真知灼见 + 真实状态  |  我们负责：钩子 + 结构 + 剪辑节奏 + 标题 + 互动引导

80%的功夫在后期。

改造对照表

现在改造后
标题 = 话题标签标题 = 一个让人想点进去的问题或反常识
开头直接讲前3秒：一句话钩子（从他的素材里提取，或加配音字幕）
讲哪算哪后期剪掉重复和跑题，保留技术精华和真实表达
没有结尾加一句总结 + 单一行动引导
B-roll很少后期穿插鸭场/鸭蛋/藻池/水培菜画面3-5处

口播结构 · 6个框架

赖博士只管讲，结构我们后期装
使用说明

赖博士拍摄时不需要看这个。拍完之后，小朱从6个框架里选一个最适合的，把他的素材装进去。

框架① 数字三分法 最常用

结构：钩子 → "三个原因/三个步骤/两个对比" → 第一…第二…第三… → 金句收

鸭场为什么不臭三个原因：①鸭粪直接入水 ②藻马上围上去 ③鱼和菜接着消纳
全程无抗怎么做到三层：①饲料配方替抗 ②水源管理 ③环境控制
DHA怎么跑到蛋里三步：①饲料添加前体 ②鸭子吸收 ③富集到蛋黄

框架② 对比法 最有攻击性

结构："你以为的"vs"实际上的" → 行业做法 vs 我们做法 → 结尾：你选哪个？

无抗的真相"市面无抗"=残留不超标 → "我们无抗"=从头到尾不用
鸭场臭不臭"你以为养鸭一定臭"→ 我们鸭场深呼吸 → 因为藻吃了鸭粪
鸭蛋不腥"普通鸭蛋"腥→做咸蛋 → "我们的"水煮直接吃

框架③ 揭秘法 好奇心驱动

结构："今天带你去看个东西"→ 现场指实物→ 讲原理→ 总结。第一视角，边走边讲。

藻工厂走到藻池边→水这么绿因为藻→每天1变8→鸭粪去哪了
鸭子排队产蛋夜里11点→卷帘门开→鸭子排着队→自动集蛋
日本分蛋机走到机器前→蛋在传送带上跑→每小时1万枚→中国只有两台

框架④ 反常识法 最易被转发

结构：抛反常识 → "很多人不知道" → 展开原理 → 颠覆认知

碳中和"鸭场不是排碳的，是吸碳的"→藻+菜吸收CO₂ > 鸭子呼出的
无粪污设施"我们没有粪污处理设施"→因为不需要→藻和鱼解决了
可生吃鸭蛋"不是所有鸭蛋都能生吃"→这颗可以→全程无抗+清洁水源

框架⑤ "只要你要 + 只要我有"法 转化型

结构：用户要什么（痛点）→ 我们有什么（方案）→ 为什么我们能做（壁垒）

给孩子买蛋你要：安全、有营养、不腥 → 我们有：全程无抗、DHA≥200、水煮直接吃
养殖户禁抗难你要：不用抗生素怎么防病 → 我们有：7年实验、饲料替抗方案

框架⑥ 一句话炸开 B类短内容

结构：最炸一句话开头 → 两个数据支撑 → 结尾引导（30-45秒）

例："这颗鸭蛋的DHA是普通鸡蛋的20倍。我们保守说8倍，实测200毫克。切开看圈纹——有三文鱼纹的就是有DHA。想吃自己来看。"

信任释放策略

TRUST RELEASE RHYTHM — 先讲什么、后讲什么、永远不讲什么
常见错误

第一条视频就讲"28年·26项专利·星火计划·新西兰"——用户还没感兴趣，全讲了也没人听。

信任五层漏斗

层级用户状态释放的信任信息示例
第1层刚刷到你只给感官记忆蛋黄圈纹 / 鸭场不臭 / 清水煮鸭不腥
第2层开始好奇给技术记忆全程无抗 / 藻工厂怎么运作 / 立体养殖原理
第3层觉得靠谱给履历记忆28年 / 26项专利 / 新西兰 / 新奥27年
第4层主动搜索主页完整信任锚点简介 + 置顶视频 = 完整IP故事
第5层想买了价格 / 检测报告 / 购买方式年卡¥2500 / DHA蛋¥188

阶段时间内容重心可以提不要讲

6.3 数据复盘体系

养殖类视频复盘的三个特殊性
维度养殖类视频特点对复盘标准的影响
受众规模垂直小众，以养殖户、准养殖户、食品关注者为主播放量基线不能和泛娱乐账号比
内容价值强实用、强时效，解决真实问题收藏、转发、私信比点赞更能反映内容价值
用户行为遇到真问题会提问，随手点赞意愿低评论率参考价值高，点赞率需适当放宽
指标冷启期基线问题诊断备注
播放量> 2000~3000低于1000：选题太窄或标题没打中痛点；1000-2000：可考虑优化前3秒养殖类过5000算正常水平，过万算表现突出
完播率> 40%低于30%：开头没讲清楚"看完能得到什么"知识类用户耐受度高，标准应高于泛娱乐
点赞率> 2%低于1%：内容没击中真实痛点或缺少情绪共鸣养殖观众点赞习惯弱，标准可放宽
收藏率> 4%高于8%：实用价值强 → 可做系列/合集养殖类收藏权重应高于点赞
评论率> 1%低于0.5%：没给用户提问或讨论的理由养殖用户遇到真问题很爱互动
转发率> 0.5%高于1%：强信任/强实用，可重点追投养殖户群聊转发价值很高
私信/咨询量按账号目标定少：结尾没有明确的引导或行动指令对卖产品/引流的账号最关键
复盘核心原则

用垂直实用账号的标准看数据，不要用泛娱乐标准。播放量低不一定是内容问题——养殖类受选题专业度、受众规模影响大。过了2000算及格，过了5000算正常，过万算表现突出。收藏和转发比点赞更能说明内容价值。
